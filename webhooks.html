Webhooks

Subscribe to real-time events and receive notifications when data changes

How Webhooks Work

Webhooks allow you to receive HTTP POST requests to your server when specific events occur in your Nexus API account.

# Your webhook endpoint will receive: { "event": "user.created", "timestamp": "2024-01-01T00:00:00Z", "data": { "id": "usr_abc123", "email": "new@user.com" } }

Available Events

user.created A new user was created
user.updated A user was updated
user.deleted A user was deleted
project.created A new project was created
project.updated A project was updated
file.uploaded A file was uploaded
file.deleted A file was deleted

Endpoints

GET /webhooks

List all webhooks

POST /webhooks

Create a new webhook

DELETE /webhooks/:id

Delete a webhook

Create Webhook

curl -X POST https://api.nexusapi.dev/v1/webhooks \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "url": "https://your-server.com/webhook", "events": ["user.created", "project.updated"], "secret": "whsec_your_secret_key" }'
💡 Tip: Always verify webhook signatures using the secret key to ensure requests are from Nexus API.

Verify Signatures

# Each webhook request includes a signature header X-Nexus-Signature: sha256=abc123... # Verify in your server: const crypto = require('crypto'); const expectedSig = crypto .createHmac('sha256', secret) .update(body) .digest('hex');
